HTTPv1 vs HTTPv2

  Firstly, the most notable difference between httpv1 & v2 is that httpv2 loads much quicker.

  HTTPv1 opens up a TCP connection for every request and does not compress it's headers. To request an HTML page, CSS stylesheet, and javascript file it takes 3 requests.

  Httpv2 provides one TCP connection to provide multiple HTTP requests for the one TCP connection. It also supports 'server-push` , the server can push multiple messages from one request to the client.

  Httpv2 also supports multiplexing, means that the server and the client can push multiple messages in parallel across a single TCP connection.
